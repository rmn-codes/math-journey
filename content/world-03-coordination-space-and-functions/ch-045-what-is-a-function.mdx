## Story Hook

Picture Gottfried Leibniz standing in his study in Hanover, Germany, in 1694. The brilliant mathematician is thinking about how to describe relationships. A sundial catches the sunlight on his desk. He watches the shadow move as the sun climbs across the sky. Every possible angle of sunlight produces exactly one position for the shadow, which tells exactly one time of day.

Leibniz writes down the word "function" for the first time in mathematical literature. He is naming something humans have always known but never formally called by name: *a rule that connects inputs to outputs in a reliable, one-to-one way*.

## Why It Matters

Functions are the language through which the universe speaks to mathematicians. A bridge engineer uses functions to show how stress distributes across a beam. A doctor uses functions to model how medicine concentration changes in the bloodstream. A climate scientist uses functions to predict temperature rises. Every single one of these describes a world where you know exactly what will happen next.

When you learn to think like a function, you gain the power to predict, to build, and to understand patterns. This is not abstract dreaming — it is how we navigate reality.

## Historical Context

Before Leibniz, mathematicians understood relationships but lacked a formal definition. The ancient Greeks used geometric proportions. Medieval scholars tracked how speed changes with time. But nobody had named this concept or created notation to capture it precisely.

Leonhard Euler, in the 1750s, refined Leibniz's idea and gave us the notation we use today: $f(x)$. This small, mighty notation changed everything. It let mathematicians build a universal language. By the 1800s, functions had become the bedrock of calculus, physics, and engineering.

## Core Concept

A **function** is a rule that assigns to each input exactly one output.

Think of it as a machine:
- You feed in a number (the input).
- The machine applies a rule.
- Out comes exactly one number (the output).

The crucial word is *exactly*. Not zero outputs. Not two outputs. Exactly one.

Here are four examples from life:

**A sundial is a function.** The input is the sun's angle in the sky. The output is the time of day. For every angle, there is exactly one time.

**A vending machine is a function.** The input is a button you press. The output is the item that drops down. Press button B7, get exactly one bag of chips (or exactly nothing if you're out of luck, but the machine is still a function — that's just the rule that day).

**A fortune-teller is NOT a function.** The input is your question. But the output changes every time you ask the same question. Sometimes you hear you'll find love. Sometimes you hear you'll find adventure. This violates the fundamental rule: one input, one output.

**A thermometer is a function.** The input is the temperature of the air. The output is the height the mercury rises. Same temperature, same height. Always.

## Definitions and Formal Lesson

**Domain:** The set of all possible inputs to a function.

If your function is "find the square root," the domain is all non-negative numbers (because you cannot take the square root of a negative number in the real numbers).

If your function is "divide 100 by this number," the domain is all numbers except zero (because you cannot divide by zero).

**Range:** The set of all possible outputs a function actually produces.

If your function is $f(x) = x^2$, and the domain is all real numbers, what is the range? The range is all non-negative numbers, because squaring any number (positive or negative) gives a non-negative result.

**Function Notation:** We write $f(x)$ to mean "the output of function $f$ when the input is $x$."

If $f(x) = 2x + 3$, then:
- $f(5) = 2(5) + 3 = 13$
- $f(0) = 2(0) + 3 = 3$
- $f(-2) = 2(-2) + 3 = -1$

The letter $f$ names the function. The letter $x$ is a placeholder for any input. You can use $g$, $h$, or $\phi$ — any letter works. What matters is that the notation shows: *this rule takes an input and produces an output*.

**The Vertical Line Test:**

To check if a graph shows a function, imagine drawing vertical lines through the graph. If every vertical line crosses the graph at most once, it's a function. If any vertical line crosses twice or more, it's not a function.

Why? Because a vertical line represents all points with the same $x$-coordinate. If a vertical line hits the graph twice, that means one input produces two outputs. That violates the definition.

## Worked Examples

**Example 1: Recognizing a Function from a Table**

Is this a function?

| Input ($x$) | Output ($y$) |
|---|---|
| 2 | 7 |
| 3 | 10 |
| 2 | 7 |
| 4 | 13 |

Yes. Every input has exactly one output. The fact that we see the input 2 twice doesn't matter — it always produces the same output (7).

**Example 2: When a Table Is Not a Function**

Is this a function?

| Input ($x$) | Output ($y$) |
|---|---|
| 1 | 5 |
| 1 | 6 |
| 2 | 8 |

No. The input 1 produces two different outputs (5 and 6). This violates the definition.

**Example 3: Using Function Notation**

Given $f(x) = -3x + 2$, find $f(-4)$.

$$f(-4) = -3(-4) + 2 = 12 + 2 = 14$$

**Example 4: Domain and Range from a Mapping Diagram**

```
Input  Output
1  →  4
2  →  5
3  →  6
```

Domain: $\{1, 2, 3\}$
Range: $\{4, 5, 6\}$

This is a function because each input points to exactly one output.

Your friend is building a machine that converts temperatures. She says:

## Application

**Temperature Conversion**

The relationship between Celsius ($C$) and Fahrenheit ($F$) is:

$$F(C) = \frac{9}{5}C + 32$$

This is a function. The input is degrees Celsius. The output is degrees Fahrenheit.

- Water freezes at $F(0) = 32°F$
- Water boils at $F(100) = 212°F$
- Room temperature: $F(20) = 68°F$

This function has domain: all real numbers (you can theoretically have any temperature), and range: all real numbers greater than $-459.67°F$ (absolute zero).

**Bank Account Balance**

Let $B(d)$ = your account balance on day $d$, starting with \$1000 and earning interest. Each day, your balance is a function of the day number. Day 1 has exactly one balance. Day 2 has exactly one balance. This is a function from "day number" to "dollars," and it governs your life.

## Recap

A **function** is a relationship where each input produces exactly one output. We name functions with letters like $f$, $g$, or $h$, and write $f(x)$ to mean "the output of $f$ when the input is $x$."

The **domain** is the set of all possible inputs. The **range** is the set of all possible outputs.

You can represent a function as a table, a graph, an equation, or a mapping diagram. To check if a graph is a function, use the **vertical line test**: if any vertical line crosses the graph more than once, it's not a function.

Functions are everywhere: thermometers, bank accounts, recipes, navigation systems, heartbeats. They are the language in which the universe encodes patterns and predictions.
