# Visual Design System

## Purpose of This Document
This document defines the visual direction for Math Journey.

The goal is not generic ed-tech cleanliness. The product should feel like a cross between an atlas, an observatory, a library, and a laboratory while remaining readable and buildable.

## Emotional Direction
The visual system should communicate:
- wonder
- seriousness
- clarity
- momentum
- historical depth

It should avoid:
- childish classroom aesthetics
- flat corporate dashboard design
- decorative chaos

## Core Visual Principles

### 1. Atmosphere with discipline
Use texture, tone, and contrast to create mood, but keep the reading experience orderly.

### 2. Content first
The page should look beautiful because the hierarchy is strong, not because every element is loud.

### 3. History and science together
The visual language should be able to hold ancient trade tablets, geometric diagrams, graphs, and physics ideas in one coherent world.

### 4. Reuse over reinvention
A small set of strong visual modules should carry most chapters.

## Layout System
The default chapter page should include:
- a strong hero or opening frame
- a readable main lesson column
- optional margin space for timeline, glossary, or progress cues
- clear separation between story, formal lesson, practice, and recap

## Reading Width
- body text should stay in a comfortably readable measure
- math blocks and diagrams may expand wider when needed
- practice sections should feel slightly more structured and modular than story sections

## Typography Direction
Use typography to signal the product’s hybrid identity:
- a serious, characterful display face for major titles
- a highly readable text face for body copy
- a disciplined math or monospace companion where technical clarity benefits from it

The type system should create contrast between:
- chapter title
- story prose
- formal definitions
- examples
- recap and assessment blocks

## Color System
The palette should feel like knowledge, materials, and sky rather than bright toy colors.

Suggested families:
- parchment, stone, or mineral neutrals for surfaces
- deep ink tones for text and frame elements
- one astronomy-inspired accent family
- one warm archival accent family
- restrained semantic colors for feedback states

## Surface Roles
At minimum, define stable visual roles for:
- page background
- card background
- elevated or highlighted card
- story panel
- theorem or formal block
- practice block
- success, warning, and error states

## Component Visual Guidance

### Story Intro
- cinematic but restrained
- distinct from the lesson body
- may use image, pattern, or historical texture

### Definition and Formula Blocks
- should feel crisp and reusable
- use strong border, background, or title treatment
- should scan quickly

### Worked Examples
- sequence should be visually obvious
- steps should feel guided, not collapsed into a wall of text

### Practice
- should feel active and task-oriented
- difficulty and progress state should be visible

### Recap and Unlock
- should signal closure and forward movement

## Diagrams and Illustration
Illustration style should:
- support comprehension first
- feel handcrafted or scholarly rather than stock-generic
- use consistent line weights and labeling

Historical imagery should feel curated, not decorative filler.

## Motion Principles
- use motion to reveal structure, not to perform personality
- page-load motion should be restrained
- step reveals should support attention and sequence
- avoid constant floating, pulsing, or ornamental animation

## Responsive Principles
The experience should work on both desktop and mobile.

That means:
- story and lesson hierarchy remain intact on small screens
- diagrams scale or stack cleanly
- practice interactions stay usable without precision pointing
- no information should exist only in hover states

## Accessibility Rules
- maintain strong contrast for text and essential controls
- do not rely on color alone for meaning
- keep type sizes generous enough for long reading
- support keyboard access for interactive patterns
- ensure formulas and diagrams can be interpreted with surrounding text

## First-Version Visual Scope
For the first version, prioritize:
- one strong chapter template
- one strong definition style
- one strong formula style
- one strong worked-example pattern
- one strong practice pattern

Do not try to build a giant design system before the first excellent chapter exists.

## Final Rule
The design should make learners feel they are entering a beautiful world of ideas while always knowing where to look next.
