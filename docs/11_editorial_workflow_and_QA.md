# Editorial Workflow and QA

## Purpose of This Document
This document defines how chapters move from idea to publishable draft.

The goal is to keep the project coherent as it scales across multiple collaborators, sessions, and tools.

## Core Workflow

### Step 1: Confirm curriculum placement
Before writing, confirm:
- world
- chapter number
- prerequisites
- chapter goal

### Step 2: Check canonical rules
Review the relevant docs for:
- learning model
- chapter schema
- voice and story style
- notation rules
- sourcing rules
- assessment rules

### Step 3: Draft metadata first
Complete the frontmatter before writing the body.

This forces clarity around:
- objectives
- level
- historical scope
- unlock expectations

### Step 4: Draft the core lesson
Write the chapter in the required order:
- story hook
- why it matters
- historical context
- concept and formal lesson
- examples
- practice
- reasoning
- application
- recap
- unlock quiz

### Step 5: Run QA passes
At minimum, review for:
- math correctness
- historical care
- voice consistency
- notation consistency
- objective alignment

### Step 6: Record resolved changes
If the work settles a project-level decision, add it to [DECISIONS.md](/Users/remonalberts/Documents/AI/math-journey/docs/DECISIONS.md).

## QA Checklist

### Mathematical QA
- definitions are correct
- formulas are correct
- examples are solved correctly
- answers in practice and quiz are correct
- notation is consistent

### Historical QA
- no invented quotes or fake certainty
- claims are appropriately scoped
- cross-cultural framing is fair
- source notes are included when needed

### Pedagogical QA
- chapter matches stated objectives
- formal difficulty fits prerequisites
- examples actually prepare the learner for the practice
- reasoning task tests understanding rather than trivia

### Voice QA
- tone is respectful
- story is vivid but controlled
- prose does not become childish or inflated
- transitions are clear

### Structural QA
- metadata is complete
- required sections are present
- recap is concise and useful
- unlock quiz checks the main learning outcomes

## Definition of a Strong Draft
A strong draft:
- could be read by a learner without outside chat context
- teaches a real concept from start to finish
- shows the intended product voice
- demonstrates the reusable chapter architecture

## First-Version Editorial Priorities
For the first version, the editorial team should prioritize:
- one excellent reference chapter
- consistency across canonical docs
- clarity over volume

## Final Rule
Publishing more chapters is not success if the underlying system becomes inconsistent. Quality and coherence come first.
