# Project Brief

## Working Title
Math Journey

## One-Sentence Description
A story-driven website that teaches mathematics and physics from basic arithmetic to AP / early college level through history, culture, formulas, theorems, worked examples, and practice problems.

## Core Idea
This product should make mathematics feel like a human adventure across civilizations rather than a disconnected sequence of school topics. Learners should move through a structured path that combines narrative, rigor, and progression.

## Primary Audience
- Curious kids and teenagers who like science, math, history, astronomy, or physics
- Students who want a deeper and more beautiful path than a standard textbook
- Curious adults restarting math from the beginning

## Learning Starting Point
The learner may only know:
- addition
- subtraction
- multiplication
- division

The platform must support a path from that level all the way to:
- high school algebra
- geometry
- trigonometry
- pre-calculus
- calculus foundations
- statistics foundations
- introductory physics concepts
- special relativity as a capstone modern topic

## Product Promise
The site will offer a comprehensive journey through mathematics and physics across time and culture, with stories that come alive and with enough rigor that learners actually master formulas, theorems, and problem solving.

## What Makes This Different
- History and culture are central, not decorative
- Math is taught as a living human project across civilizations
- Every chapter combines story, formal learning, and practice
- Learners progress through a clear sequence with unlocks and milestones
- The experience should feel intelligent, cinematic, and respectful, not childish

## Product Format
The product will likely combine:
- a website as the main learning platform
- chapter-based progression
- practice and quizzes
- unlockable next chapters
- a possible newsletter cadence for releases
- optional interactive diagrams, simulations, and visual tools

## Educational Philosophy
The platform should teach in four layers:
1. Intuition
2. Mechanics
3. Reasoning
4. Application

Learners should not only compute answers, but also understand where ideas came from, why they mattered, and how they connect to the real world.

## Historical Philosophy
The site should not present math as a narrow chain of isolated European geniuses. It should show mathematics as a long, cross-cultural construction involving many civilizations, including but not limited to:
- Mesopotamia
- Egypt
- Greece
- India
- the Islamic world
- Europe
- later global scientific developments

## Experience Goals
The learner should feel:
- curiosity
- momentum
- intellectual challenge
- historical wonder
- growing mastery

The site should feel like:
- part atlas
- part observatory
- part library
- part laboratory
- part rigorous course

## Core Content Structure
Each chapter should eventually include:
- a story hook
- historical context
- explanation of why the idea mattered
- core definitions
- formulas and theorems
- worked examples
- practice problems
- a reasoning or proof task
- a real-world or physics connection
- a recap or unlock quiz

## Scope
This project is intended to grow into a large, comprehensive curriculum. It should be designed from the beginning as a system, not as a loose set of articles.

## Initial Build Strategy
Start small and structured:
- define the curriculum and rules
- create canonical documentation
- build one excellent reference chapter
- then expand into a reusable content and design system

## Immediate Priorities
1. Define the curriculum map
2. Define the learning model
3. Define the chapter schema
4. Define the interaction patterns
5. Create the first sample chapter

## Non-Goals for the First Version
- building the entire curriculum at once
- overcomplicated gamification
- too many custom interaction types
- a broad teacher dashboard
- full social features
- advanced personalization

## Success for the First Version
The first version succeeds if:
- the concept is coherent
- the tone is compelling
- the math is rigorous and clear
- the structure can scale
- one or more sample chapters prove the model works

## Working Principle
The source of truth should live in files, not in chat history. Important decisions should be captured in markdown documents so the project can scale cleanly across ChatGPT, Codex, and the repo.
