# Math Notation and Glossary Rules

## Purpose of This Document
This document defines how mathematical language, notation, and glossary terms should be introduced and used across Math Journey.

The goal is clarity, consistency, and gradual rigor.

## Core Notation Principles

### Principle 1: Meaning before symbolism
Introduce the idea first, then the notation.

### Principle 2: Define every symbol
If a symbol appears in instructional content, the learner should be able to identify what it means from the local context.

### Principle 3: One role per symbol
Avoid reusing the same letter for different quantities in the same example unless there is a compelling reason.

### Principle 4: Use notation only when it helps
Do not introduce formal notation just to look advanced.

### Principle 5: Keep notation stable
Once a notation convention is established in a chapter or system, do not switch casually.

## Formatting Rules
- inline symbols should appear in backticks when discussed as notation in prose, for example `x`, `+`, or `=`
- displayed equations should be visually separated from the surrounding prose in the eventual product
- variable names should be accompanied by a short meaning statement when first introduced
- units should be explicit whenever a quantity represents measurement

## Order of Introduction
For a new symbol or expression, prefer this sequence:
1. plain-language idea
2. named quantity
3. notation
4. worked use

Example pattern:
- "A quantity can be unknown."
- "We will call that unknown amount the variable."
- "We write it as `x`."
- "If `x + 3 = 10`, then `x = 7`."

## Rules for Numbers and Operations
- write small whole-number examples plainly unless symbolic emphasis is needed
- use parentheses whenever omission could confuse a beginner
- avoid compact but advanced notation in early worlds
- prefer explicit steps in worked examples over compressed algebra

## Rules for Formulas
Every formula presentation should include:
- the formula itself
- the meaning of each symbol
- a short plain-language interpretation
- any important constraint or condition

Do not present formulas as isolated decorations.

## Equality and Equivalence
Early chapters should be especially careful with symbols such as:
- `=`
- `<`
- `>`
- `≈`

The product should distinguish:
- exact equality
- inequality
- approximation
- definition or naming language in prose

## Function Notation
Function notation should be delayed until the learner is ready for it.

When introduced:
- explain that `f(x)` names the output of a function for input `x`
- do not assume the parentheses mean multiplication
- contrast function notation with ordinary products when confusion is likely

## Geometry and Diagram Labels
- diagram labels should be consistent with the text
- points should typically use capital letters
- lengths, angles, and variables should be introduced explicitly
- diagrams should not contain unexplained symbols

## Notation Load by Level

### Foundational chapters
- low notation density
- more words
- more repeated reminders of symbol meaning

### Core chapters
- standard school and textbook notation
- more efficient symbolic manipulation

### Advanced and capstone chapters
- denser notation is acceptable, but definitions must still be local and explicit

## Glossary Rules
The glossary should be cumulative and learner-facing.

Each glossary entry should include:
- term
- short definition
- plain-language restatement when helpful
- chapter or world context
- related terms

## Glossary Entry Style
A strong glossary entry is:
- short
- exact
- non-circular
- readable by the intended audience

Bad glossary pattern:
- defining a term only with more advanced undefined terms

## Synonyms and Naming
When multiple names exist:
- introduce the most learner-useful name first
- note important synonyms
- mention historical or regional variation only when it helps understanding

## Pronunciation and Etymology
These are optional.

Include them when:
- the word is likely to intimidate learners
- the origin helps memory
- the term appears often across many chapters

## First-Version Rules
For the first version:
- prioritize consistency over clever notation choices
- use the most standard school-compatible notation unless there is a strong pedagogical reason not to
- include a small glossary or key-term list in every core chapter

## Final Rule
Notation should make the idea clearer, not more prestigious.
